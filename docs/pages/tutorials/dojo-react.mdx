# Dojo React

Let's build a basic counter React application using Dojo.

:::note
This tutorial will guide you through building a React client for your Dojo world.
You should have some familiarity with React and TypeScript.
If you're new to Dojo, we recommend starting with the [Dojo Starter tutorial](/tutorials/dojo-starter).
:::

## What we're building

We will build a simple counter application where:

1. Players can increase and decrease a global counter
2. Each player has their own moves counter
3. The UI updates in real-time using subscriptions

<Steps>

### Setup the Dojo project

First, ensure you have Dojo installed by following the [installation guide](/installation).

Create a new Dojo project:

```bash
dojoup -v 1.0.0-rc.6
sozo init dojo-react-tutorial
cd dojo-react-tutorial
```

### Create the Counter Model

Let's start by defining our game state.
Replace the contents of `src/models.cairo`:

```rust
use starknet::ContractAddress;

#[derive(Copy, Drop, Serde)]
#[dojo::model]
pub struct Counter {
    #[key]
    pub counter_id: u32,
    pub value: u32,
}

#[derive(Copy, Drop, Serde)]
#[dojo::model]
pub struct Moves {
    #[key]
    pub player: ContractAddress,
    pub remaining: u8,
    pub last_direction: Direction,
}

#[derive(Serde, Copy, Drop, Introspect, PartialEq)]
pub enum Direction {
    None,
    Up,
    Down,
}

impl DirectionIntoFelt252 of Into<Direction, felt252> {
    fn into(self: Direction) -> felt252 {
        match self {
            Direction::None => 0,
            Direction::Up => 1,
            Direction::Down => 2,
        }
    }
}
```

### Create the Actions System

Now let's create the system that will handle our counter logic.
Replace the contents of `src/actions.cairo`:

```rust
use dojo_react_tutorial::models::{Counter, Moves, Direction};
use starknet::{ContractAddress, get_caller_address};

#[dojo::interface]
trait IActions {
    fn spawn();
    fn increment();
    fn decrement();
}

#[dojo::contract]
mod actions {
    use super::{IActions, Counter, Moves, Direction, ContractAddress, get_caller_address};

    #[abi(embed_v0)]
    impl ActionsImpl of IActions<ContractState> {
        fn spawn(world: @IWorldDispatcher) {
            let player = get_caller_address();

            set!(
                world,
                (
                    Moves {
                        player,
                        remaining: 10,
                        last_direction: Direction::None,
                    },
                    Counter { counter_id: 0, value: 0 },
                )
            );
        }

        fn increment(world: @IWorldDispatcher) {
            let player = get_caller_address();

            let mut moves = get!(world, player, (Moves));
            assert(moves.remaining > 0, 'no moves remaining');

            let mut counter = get!(world, 0, (Counter));
            counter.value += 1;

            moves.remaining -= 1;
            moves.last_direction = Direction::Up;

            set!(world, (moves, counter));
        }

        fn decrement(world: @IWorldDispatcher) {
            let player = get_caller_address();

            let mut moves = get!(world, player, (Moves));
            assert(moves.remaining > 0, 'no moves remaining');

            let mut counter = get!(world, 0, (Counter));
            counter.value -= 1;

            moves.remaining -= 1;
            moves.last_direction = Direction::Down;

            set!(world, (moves, counter));
        }
    }
}
```

### Build and Deploy

Build the project:

```bash
sozo build
```

Start Katana (our local testnet):

```bash
katana --disable-fee --allowed-origins "*"
```

In another terminal, migrate the world:

```bash
sozo migrate apply
```

This will deploy our world and contracts to Katana.


Install the Dojo SDK:


First, we need to generate TypeScript bindings for our Dojo models and systems.
From the project root (not the client directory):

```bash
sozo build --typescript
```

This creates TypeScript definitions in `typescript/` directory.

### Setup Dojo Provider

Create `client/src/DojoContext.tsx`:

```tsx
import React, { createContext, ReactNode, useContext } from "react";
import { BurnerManager, useBurnerManager } from "@dojoengine/react";
import { Account } from "starknet";

interface DojoContextType {
    account: Account;
    burnerManager: BurnerManager;
}

const DojoContext = createContext<DojoContextType | undefined>(undefined);

export const useDojo = () => {
    const context = useContext(DojoContext);
    if (!context) {
        throw new Error("useDojo must be used within a DojoProvider");
    }
    return context;
};

interface DojoProviderProps {
    children: ReactNode;
}

export const DojoProvider = ({ children }: DojoProviderProps) => {
    const burnerManager = useBurnerManager({
        burnerManagerOptions: {
            rpcProvider: "http://localhost:5050",
        },
    });

    return (
        <DojoContext.Provider
            value={{
                account: burnerManager.account as Account,
                burnerManager,
            }}
        >
            {children}
        </DojoContext.Provider>
    );
};
```

## Understanding the Architecture

### Models

Our Dojo models define the shape of our game state:

- `Counter`: Stores the global counter value
- `Moves`: Tracks each player's remaining moves and last action

### Systems

The `actions` contract contains our game logic:

- `spawn`: Initializes a new player with 10 moves
- `increment`/`decrement`: Modifies the counter and updates player state

### React Integration

The React app uses:

- **DojoContext**: Manages account and connection state
- **Burner Manager**: Handles temporary accounts for testing
- **Direct Contract Calls**: Executes transactions on our world

## Next Steps

This tutorial showed you the basics of building with Dojo and React.
To build a production application, consider:

1. **Using Torii**: Set up a GraphQL indexer for efficient data fetching
2. **Real-time Updates**: Implement subscriptions for live state updates
3. **Generated SDK**: Use `sozo build --typescript` output for type-safe contract interactions
4. **Better State Management**: Consider using React Query or similar for caching
5. **Error Handling**: Add comprehensive error handling and user feedback

For more advanced examples, check out the [onchain chess tutorial](/tutorials/onchain-chess/README) which demonstrates more complex game mechanics and state management.
